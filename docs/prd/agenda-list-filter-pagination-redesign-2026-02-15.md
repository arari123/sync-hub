# PRD: 이슈(안건) 관리 필터/페이지네이션 UI 개편 (2026-02-15)

## 목적
- 이슈(안건) 관리 페이지의 필터링 패널을 메인 페이지 프로젝트 리스트 필터 패널과 동일한 톤으로 통일한다.
- 이슈(안건) 관리 페이지의 페이지 이동 버튼을 메인 페이지 프로젝트 리스트 페이지네이션과 동일한 위치/디자인으로 통일한다.

## 범위
- 프론트엔드: `frontend/src/pages/AgendaList.jsx`

## 요구사항
- REQ-001: 필터링 패널을 메인 페이지 프로젝트 리스트 필터링 패널과 동일한 디자인 톤으로 변경한다.
- REQ-002: 검색어 입력창 폭을 좁히고, 입력창 왼쪽에 `검색 조건` 드롭다운을 배치한다.
- REQ-003: 검색어 입력창 오른쪽에 상태/유형/임시저장 포함 등의 필터 버튼(칩)을 배치한다.
- REQ-004: `표시 개수`(per-page) 필터링 UI를 제거한다.
- REQ-005: 페이지 이동 버튼(맨앞/이전/숫자/다음/맨뒤)을 메인 페이지 프로젝트 리스트와 동일한 위치/디자인으로 적용한다.

## 수용 기준
- AC-001: 필터 패널이 메인 페이지의 필터 패널과 동일한 스타일(`chip`, `app-surface-soft`)로 표시된다.
- AC-002: 검색 조건 드롭다운 + 검색어 입력창 + 필터 칩이 한 줄(데스크탑 기준)로 정렬된다.
- AC-003: 표시 개수 선택 UI가 더 이상 노출되지 않는다.
- AC-004: 페이지네이션이 10개 단위 숫자 페이지와 맨앞/맨뒤 버튼을 포함하며, 디자인이 메인 페이지와 동일하다.

- `docker exec -w /app synchub_frontend npm run build`
