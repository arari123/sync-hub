# PRD: 프로젝트 설정 페이지 삭제 버튼 추가 (2026-02-18)

## 배경
- 프로젝트 설정 페이지에서 프로젝트를 삭제할 수 있는 기능이 없어, 오생성 프로젝트 정리가 어렵다.

## 목표
- 프로젝트 설정 페이지에서 삭제 버튼을 제공하고, 권한/데이터 일관성을 지키며 프로젝트를 삭제할 수 있게 한다.

## 요구사항
- REQ-001: 설정 페이지(`BudgetProjectInfoEdit`)에 `프로젝트 삭제` 버튼을 노출한다.
- REQ-002: 삭제 버튼 클릭 시 사용자 확인 절차를 거친다.
- REQ-003: 백엔드에 `DELETE /budget/projects/{project_id}` API를 제공한다.
- REQ-004: 삭제 권한은 기존 수정 권한 정책(`can_edit`)을 따른다.
- REQ-005: 삭제 시 프로젝트 예산 버전/설비 데이터는 함께 정리한다.
- REQ-006: 문서 연결(`documents.project_id`)은 `null`로 해제한다.
- REQ-007: 하위 AS 프로젝트 또는 안건 데이터가 있는 프로젝트는 삭제를 차단하고 명확한 오류를 반환한다.

## 완료 기준
- 로컬 Docker 기준 `verify:fast` 통과
- 설정 페이지에서 삭제 버튼 클릭 후 성공/실패 케이스 메시지 확인 가능
