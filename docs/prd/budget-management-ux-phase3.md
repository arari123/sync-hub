# Budget Management UX Phase3 PRD

## 목적
예산/집행 입력 UX를 프로젝트 상태 기반으로 단순화하고, 인건비/경비 입력을 자동화하여 입력 실수를 줄인다.

## 요구사항
1. 예산 입력창과 집행 입력창은 분리되어야 하며, 프로젝트 상태에 따라 자동 전환된다.
2. 인건비/경비의 국내/해외 설치 구분은 프로젝트 기본 정보(설치 장소) 기반으로 자동 결정된다.
3. 입력 페이지에서는 국내/해외 전환 버튼을 제거하고, 현재 적용 기준(국내/해외 및 국가/장소)을 표시한다.
4. 인건비의 부서 설정 입력 UI(콤마 입력)는 제거한다.
5. 인건비는 빠른 추가 방식으로 부서를 행에 추가할 수 있어야 한다.
   - 자체 인원: 설정된 기본 부서 목록
   - 외주 인원: PM, 설계, 기구, 전장, 제어, SW
   - 동일 부서를 여러 번 추가하면 `부서명(2)`, `부서명(3)` 형태로 자동 생성
6. 인건비 행에는 자체/외주 구분이 저장되어야 한다.
7. 경비 자동 산정은 기본 자동 적용되어야 하며, 재료비/인건비 변경 시 자동 업데이트된다.
8. 경비 자동 산정 버튼은 초기화 용도로 동작해야 한다.
9. 경비 자동항목 중 특정 항목을 잠금하면 초기화/자동갱신에서 제외되어야 한다.

## 범위
- 프론트엔드 `BudgetProjectEditor` UX/동작 개편
- 데이터 구조 확장(`staffing_type`, `lock_auto`)
- 백엔드 스키마 변경 없음(JSON detail 확장 사용)

## 비범위
- 별도 부서 설정 페이지 구현
- 설치 국가 전용 입력 필드 신규 추가
