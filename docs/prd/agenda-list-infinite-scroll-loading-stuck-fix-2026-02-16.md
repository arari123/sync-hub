# PRD: 안건 리스트 인피니티 스크롤 로딩 고정 버그 수정 (2026-02-16)

## 목적
- 검색어 입력 후 스크롤 하단 로드 시 `더 불러오는 중...` 상태가 고정되고 추가 목록이 갱신되지 않는 문제를 해결한다.

## 적용 범위
- `frontend/src/components/agenda/AgendaSplitView.jsx`

## 문제 정의
- 검색 조건 변경 중 기존 추가 로드 요청(page>1)이 취소될 때, 로딩 상태(`isLoadingMore`)와 락(`loadMoreLockRef`) 해제가 누락될 수 있다.
- 그 결과 리스트가 계속 로딩 중 상태로 남아 추가 로드가 진행되지 않는다.

## 요구사항
- REQ-001: 취소된 추가 로드 요청(page>1)에서도 로딩 상태가 해제되어야 한다.
- REQ-002: 취소된 추가 로드 요청(page>1)에서도 다음 로드를 막는 락이 해제되어야 한다.
- REQ-003: 검색 후 스크롤 시 추가 목록이 정상 갱신되어야 한다.

## 수용 기준
- AC-001: 검색 입력 후 스크롤 하단 도달 시 로딩 텍스트가 고정되지 않는다.
- AC-002: 같은 화면에서 추가 10개 로드가 정상 반복된다.
