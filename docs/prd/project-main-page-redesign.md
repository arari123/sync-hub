# 프로젝트 메인 페이지 개편 PRD

## 배경
- 기존 `/project-management/projects/:projectId` 페이지는 프로젝트 상세 대시보드 형태이며, 메인 검색 페이지의 상단바/브레드크럼 체계와 시각적으로 분리되어 있다.
- `docs/ex project main.html` 기반의 레이아웃을 프로젝트 메인 화면에 적용해야 한다.

## 목표
- '프로젝트 상세' 명칭을 '프로젝트 메인'으로 변경한다.
- 프로젝트 메인 페이지에 메인 페이지 상단바를 유지한다.
- 본문을 `docs/ex project main.html` 클론 스타일로 개편하되 실제 프로젝트 데이터를 표시한다.

## 요구사항
1. 명칭 변경
- 사용자 노출 텍스트의 '프로젝트 상세'를 '프로젝트 메인'으로 변경한다.

2. 상단 구조
- 메인 페이지 상단바(로고/검색/알림/메뉴/유저)를 프로젝트 메인 페이지에서도 유지한다.
- 브레드크럼은 상단바 하단 패널에 표시한다.
- 경로 형식: `메인 / 글로벌 검색 > (프로젝트 이름)`
- 프로젝트 이름은 10자 이하로 표시한다.

3. 본문 레이아웃
- `docs/ex project main.html`의 레이아웃을 클론해 적용한다(브레드크럼 블록은 제외).
- 실제 데이터 바인딩:
  - 프로젝트명, 단계, 설명, 고객사, 설치 위치, 담당자, 프로젝트 코드
  - `ERP-CODE`는 프로젝트 코드 사용
- Timeline 마일스톤은 임의 일정으로 표시한다(향후 일정 기능 연동 전제).
- Latest issues는 빈 상태 레이아웃만 유지한다.

## 완료 기준
- `/project-management/projects/:projectId` 페이지가 '프로젝트 메인' 기준 UI로 동작한다.
- 브레드크럼 경로/표시 길이 규칙이 적용된다.
- Docker 환경에서 `verify:fast`가 통과한다.
