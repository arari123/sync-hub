# PRD: 일정 마일스톤 라벨 겹침 및 집행 입력 행 시드 보강 (2026-02-19)

## 배경
- 일정 관리의 마일스톤 패널에서 이벤트 날짜가 근접하면 라벨이 겹쳐 가독성이 저하된다.
- 집행금액 입력 모드에서 인건비/경비 항목이 비어 있어 집행금액을 입력하기 어려운 케이스가 있다.

## 목표
- 마일스톤 이벤트 라벨이 근접 날짜에서도 겹치지 않게 배치한다.
- 집행금액 입력 모드에서 인건비/경비가 즉시 입력 가능한 상태로 시작되게 한다.

## 범위
- 프론트엔드
  - `BudgetProjectScheduleManagement`: 이벤트 라벨 충돌 회피 및 행 높이 자동 확장
  - `BudgetProjectEditor`: 집행 모드 인건비 항목 추가 UX 활성화, 경비 기본 항목 자동 시드

## 비범위
- 일정 데이터 모델/API 변경
- 예산/집행 계산 공식 변경

## 요구사항
- REQ-001: 마일스톤 이벤트 라벨은 근접 날짜에서도 같은 lane에 겹치지 않게 배치되어야 한다.
- REQ-002: 라벨 lane 수가 늘면 단계 행 높이가 자동으로 확장되어 라벨이 잘리지 않아야 한다.
- REQ-003: 집행 모드 인건비 탭에서도 부서 버튼으로 항목을 추가할 수 있어야 한다.
- REQ-004: 집행 모드 경비 탭은 기본 항목이 자동 표시되어 집행금액을 바로 입력할 수 있어야 한다.
- REQ-005: Docker 환경 `verify:fast` 검증이 통과해야 한다.
