# PRD: 일정관리 조회 마일스톤 정렬 및 필터 패널 디자인 통일

## 목적
- `일정 관리 > 일정 조회` 화면에서 상단 마일스톤 패널이 하단 간트차트의 `일정 열(좌측 컬럼)` 및 타임라인과 시각적으로 정렬되도록 개선한다.
- 동일 화면의 필터링 패널을 메인 페이지(프로젝트 리스트) 필터 패널과 동일한 UI 톤(칩 기반, 컴팩트 입력, 반응형 구성)으로 통일한다.

## 요구사항

### SCH-UX-010: 마일스톤 패널 정렬
- [AC-1] `/project-management/projects/:projectId/schedule`에서 마일스톤 타임라인은 간트차트 타임라인과 동일한 기준으로 시작해야 한다.
  - 즉, 간트차트의 좌측 `일정 열(고정폭 460px)`만큼 마일스톤 영역도 동일한 좌측 컬럼을 확보해야 한다.
- [AC-2] 마일스톤 패널은 간트차트와 동일한 최소 폭(스크롤 기준)을 적용하여 날짜 tick 및 이벤트 포인트가 동일한 비율로 렌더링되어야 한다.
- [AC-3] 사용자가 마일스톤 패널 또는 간트차트의 가로 스크롤을 이동하면 두 영역의 가로 스크롤 위치가 동일하게 유지되어야 한다.

### SCH-UX-012: 마일스톤 패널 단계 요약 바 및 이벤트 표시
- [AC-1] 마일스톤 패널에는 단계(설계/제작/설치)별 요약 진행 바가 3줄로 표시되어야 한다.
- [AC-2] 각 단계 요약 바의 색상은 하단 간트차트(단계 색상)와 동일해야 한다.
- [AC-3] 이벤트는 하단 간트차트에서 별도 행으로 표시하지 않아도 되며, 마일스톤 패널의 해당 단계 요약 바 위에 동그란 아이콘으로 오버레이되어야 한다.
- [AC-4] 마일스톤 패널 좌측에는 단계 명칭과 해당 단계의 시작/종료 날짜가 표시되어야 한다.
- [AC-5] 마일스톤 패널의 제목(헤더) 영역은 세로폭을 최소화해야 한다.

### SCH-UX-011: 필터 패널 디자인 통일
- [AC-1] 일정 조회 화면의 필터 UI는 메인 페이지 프로젝트 리스트 필터 패널과 동일한 디자인 토큰(칩, 입력, 구분선, 포커스 링)을 사용해야 한다.
- [AC-2] 단계/구분/상태 필터는 `select`가 아닌 `칩 버튼`으로 제공해야 한다.
- [AC-3] 모바일에서는 필터 토글 버튼으로 필터 영역을 접고 펼칠 수 있어야 한다.
- [AC-4] 기존 필터 로직(검색어 + 단계/구분/상태 필터)의 동작은 동일해야 한다.
