# PRD: 안건 페이지 DB 연동 및 실데이터 전환

## 목적
기존 `agenda` 화면(목업)을 실제 DB 기반 기능으로 전환하여, 프로젝트별 안건 작성/조회/답변/코멘트/검색이 가능하도록 구현한다.

## 요구사항

### AGD-REQ-001: 안건 유형 및 등록 흐름
- [AC-1] 안건은 `일반 안건`과 `작업보고서` 유형을 지원한다.
- [AC-2] 안건 작성 시 `등록`과 `임시저장`을 지원한다.
- [AC-3] 임시저장 데이터는 작성자가 이후 다시 불러와 수정 후 정식 등록할 수 있다.
- [AC-4] 모든 안건은 생성 시 코드명이 자동 생성된다.

### AGD-REQ-002: 안건 데이터 모델
- [AC-1] 최초 안건/답변 안건/추가 작업보고서를 저장할 수 있는 엔티티 구조를 제공한다.
- [AC-2] 작성자/요청자/답변자/소속, 제목/본문, 상태, 첨부파일, 작업자/작업시간/파츠 등 필드를 저장한다.
- [AC-3] 코멘트는 안건 스레드 단위로 저장한다.
- [AC-4] 정식 등록된 안건/코멘트는 수정/삭제 API를 제공하지 않는다.

### AGD-REQ-003: 목록(F) 화면
- [AC-1] 프로젝트 안건 목록은 실데이터 기반으로 조회된다.
- [AC-2] 정렬은 `진행 중 우선` + `최신 업데이트 순`을 따른다.
- [AC-3] 검색(공백/콤마 분리 단어 매칭), 조건 검색, 상태 필터, 페이지당 10/30/50, 페이지네이션을 지원한다.
- [AC-4] 목록 카드에 유형, 제목(최초/최신), 인원 정보, 상태, 첨부 수, 답변 수, 코멘트 수를 표시한다.

### AGD-REQ-004: 상세(O) 화면
- [AC-1] 최초 안건과 최신 답변은 펼쳐진 상태로 표시한다.
- [AC-2] 중간 답변은 제목 중심 접힘 상태로 표시하며 개별 펼치기/전체 펼치기를 지원한다.
- [AC-3] 코멘트 작성 및 최신순 조회를 지원한다.
- [AC-4] 진행중/완료 상태 변경은 최초 등록자만 가능하다.

### AGD-REQ-005: 작성(P) 화면 UX
- [AC-1] 본문은 리치 텍스트 편집(폰트 크기/색상/밑줄/표 삽입 등 기본 기능)을 제공한다.
- [AC-2] 클립보드 이미지 붙여넣기 삽입을 지원한다.
- [AC-3] 첨부파일 업로드를 지원한다.

## 범위
- 백엔드: 안건 테이블, API, 파일 업로드/조회
- 프론트엔드: `AgendaList`, `AgendaCreate`, `AgendaDetail` 실데이터 연동
- 문서: PRD, 실행계획

## 비범위
- 외부 문서 OCR/벡터 검색과 안건의 자동 연계
- 완전한 WYSIWYG 고급 기능(협업 실시간 편집 등)
