# PRD: 이슈(안건) 리스트 답변 분할 표시 및 등록 확인창

## 목적
- `이슈 관리(안건 관리)` 목록에서 답변이 존재하는 안건은 최초 등록 내용과 최신 답변 내용을 한 눈에 비교할 수 있도록 리스트 카드 UI를 2분할로 제공한다.
- 안건/답변 등록 시 실수 등록을 방지하기 위해 확인창을 추가한다.

## 요구사항

### AGD-LIST-001: 답변 존재 시 2분할 카드
- [AC-1] `/project-management/projects/:projectId/agenda`(안건 관리) 목록에서 `reply_count > 0`인 안건은 카드 본문이 2분할로 표시되어야 한다.
- [AC-2] 좌측 패널에는 `최초 등록 안건`(root) 정보를 표시해야 한다.
  - 제목, 요약(plain text), 작성자, 등록일을 포함한다.
- [AC-3] 우측 패널에는 `최신 답변 안건`(latest reply) 정보를 표시해야 한다.
  - 제목, 요약(plain text), 최종 작성자, 최신 업데이트 일시를 포함한다.
- [AC-4] 답변이 없는 안건은 기존 단일 카드 레이아웃을 유지한다.

### AGD-REG-001: 안건 등록 확인창
- [AC-1] `/project-management/projects/:projectId/agenda/new`에서 `등록`(published) 버튼 클릭 시 `안건을 등록하시겠습니까?` 확인창이 표시되어야 한다.
- [AC-2] 사용자가 취소를 선택하면 등록 API 호출이 발생하지 않아야 한다.

### AGD-REG-002: 답변 등록 확인창
- [AC-1] `/project-management/projects/:projectId/agenda/:agendaId`에서 `답변 등록` 버튼 클릭 시 `답변을 등록하시겠습니까?` 확인창이 표시되어야 한다.
- [AC-2] 사용자가 취소를 선택하면 등록 API 호출이 발생하지 않아야 한다.

## 범위
- 프론트: 안건 목록 카드 UI, 안건/답변 등록 confirm 처리
- 백엔드: 안건 목록 API 응답에 root/latest 요약(plain) 필드 추가

## 비범위
- 안건/답변 데이터 모델 변경
- 권한 정책 변경
