# PRD: 데모 데이터 재구성 + 설계 단계 추가 (2026-02-15)

## 배경
메인(`/home`)의 프로젝트 단계 필터에서 `설계`가 누락되어 있고, 테스트/데모를 위해 프로젝트/안건/예산/일정 데이터가 충분히 준비되어 있지 않다.

## 목표
1. 메인(`/home`) 프로젝트 단계 필터에 `설계`를 포함한다.
2. 프로젝트 진행 단계(`current_stage`)에 `design(설계)`를 추가해, 설비 프로젝트의 기본 단계(설계-제작-설치-워런티)를 자연스럽게 표현한다.
3. 기존 프로젝트/안건 데이터를 모두 삭제하고, 다양한 케이스를 포함한 데모 데이터를 자동으로 시드한다.

## 용어/규칙
- 프로젝트 종류(project_type): `equipment(설비)`, `parts(파츠)`, `as(AS)`
- 설비 프로젝트 일정 단계(타임라인/마일스톤): `설계-제작-설치-워런티`
- `워런티`는 설비 프로젝트의 일정 단계 명칭이며, 프로젝트 종류 명칭이 아니다.

## 요구사항
### FE-REQ
- FE-REQ-001: 메인(`/home`) 프로젝트 단계 필터 버튼에 `설계`가 표시되어야 한다.

### BE-REQ
- BE-REQ-001: 프로젝트 단계 정규화/라벨링에 `design(설계)`를 추가한다.
- BE-REQ-002: 프로젝트 단계 정렬(랭크) 로직에 `design`을 포함한다.

### SEED-REQ
- SEED-REQ-001: 기존 프로젝트/안건(예산/일정 포함) 데이터를 일괄 삭제한다.
- SEED-REQ-002: 임의 프로젝트 약 30개를 생성한다.
- SEED-REQ-003: 각 프로젝트에 안건 20개를 생성한다.
  - 안건 본문은 10~20줄 이상이며, 각 안건 내용은 서로 달라야 한다.
- SEED-REQ-004: 각 프로젝트에 예산 데이터가 입력되어 있어야 한다.
- SEED-REQ-005: 각 프로젝트에 일정 데이터가 입력되어 있어야 한다.
- SEED-REQ-006: 가능한 한 다양한 입력 케이스를 포함한다(프로젝트 종류/단계/예산 항목/집행/일정 주말 모드 등).

## 수용 기준(AC)
1. `/home` 단계 필터에 `설계` 버튼이 표시된다.
2. `current_stage=design` 프로젝트가 정상적으로 라벨링되어 `설계`로 표시된다.
3. 시드 스크립트 실행 후 프로젝트 약 30개, 프로젝트당 안건 20개가 생성된다.
4. 시드된 프로젝트는 예산/일정 데이터가 조회 화면에서 표시된다.
5. Docker 환경에서 `verify_fast` 및 프론트 `build`가 통과한다.
