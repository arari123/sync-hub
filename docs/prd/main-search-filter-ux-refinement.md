# PRD: 메인 검색 페이지 필터/패널 UX 정리

## 목적
메인 검색 페이지에서 프로젝트 중심 UX를 강화한다. 메인 페이지 사이드바 메뉴를 제거하고, 프로젝트/필터 패널 노출 조건 및 필터 조작 방식을 개선한다.

## 요구사항

### MSFR-REQ-001: 메인 사이드바 정리
- [AC-1] 메인 페이지 사이드바 영역의 메뉴를 모두 제거한다.
- [AC-2] 다른 페이지 사이드바 구조에는 영향을 주지 않는다.

### MSFR-REQ-002: 프로젝트 패널 조건부 노출
- [AC-1] 검색 결과에 프로젝트가 매칭되지 않으면 프로젝트 패널을 숨긴다.
- [AC-2] 페이지 접속 시 프로젝트가 전혀 없으면 프로젝트 패널을 숨긴다.

### MSFR-REQ-003: 필터 패널 개선
- [AC-1] 필터 패널은 기본 펼침 상태여야 한다.
- [AC-2] 필터 패널은 프로젝트가 있을 때만 노출한다.
- [AC-3] 필터 패널 명칭을 `프로젝트 필터`로 변경한다.
- [AC-4] 프로젝트명/코드/고객사/담당자 텍스트 입력은 제거한다.
- [AC-5] 내 프로젝트/전체 프로젝트 토글을 필터 영역으로 이동한다.
- [AC-6] 프로젝트 생성 버튼은 필터 영역에 어울리는 스타일로 변경한다.
- [AC-7] 단계 필터는 드롭다운이 아닌 복수 선택 버튼 UI로 제공한다.

### MSFR-REQ-004: 문서 패널 유지 정책
- [AC-1] 문서 검색 결과 패널은 검색 실행 시에만 노출한다.
