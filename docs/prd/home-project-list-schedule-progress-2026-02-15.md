# PRD: 메인 프로젝트 리스트 단계 일정 프로그레스바 (2026-02-15)

## 배경
메인(`/home`)의 프로젝트 리스트는 현재 진행률(퍼센트)만 표시해 단계별 일정(설계/제작/설치/AS/종료)을 한눈에 파악하기 어렵다. 또한 `검토` 단계 프로젝트는 현재 프로그레스바 상에서 맥락이 약하다.

## 목표
1. 프로젝트 리스트의 프로그레스바를 **설계-제작-설치-AS** 4단계로 구분해 표시한다.
2. 각 단계에 대해 **일정관리(WBS 일정)** 과 연동된 날짜 정보를 함께 표시한다.
3. `검토` 단계 프로젝트는 프로그레스바 위에 오버레이로 **검토 단계 + 프로젝트 생성일자**를 표시한다.
4. `종료` 단계 프로젝트는 프로그레스바 위에 오버레이로 **종료 단계 + 종료일자**를 표시한다.

## 비목표
- 일정관리(WBS) 데이터 구조/그룹을 변경하지 않는다.
- 리뷰(검토) 단계의 별도 일정 그룹을 추가하지 않는다.

## 요구사항
### FE-REQ
- FE-REQ-001: 메인(`/home`)의 프로젝트 리스트에서 단계 프로그레스바는 `설계-제작-설치-AS` 4단계로 표시한다.
- FE-REQ-002: 단계별 날짜는 일정관리(`GET /budget/projects/{project_id}/schedule`)의 WBS 데이터를 기반으로 계산한다.
  - 설계/제작/설치: 각 단계 root 그룹(`stage-design`, `stage-fabrication`, `stage-installation`)의 `first_start ~ last_end`를 표시한다.
  - AS: `설치`의 `last_end`를 AS 시작일로 사용한다.
  - 종료: AS 시작일의 1년 후 날짜로 표시한다.
- FE-REQ-002-1: 단계별 날짜는 **3행(단계명/시작/끝)** 으로 표시해 잘리지 않도록 한다.
- FE-REQ-002-2: 하단 날짜 영역에는 `종료` 단계 라벨/날짜를 표시하지 않는다.
- FE-REQ-003: `검토` 단계 프로젝트는 프로그레스바 위 오버레이로 `검토 단계 · 생성 YYYY.MM.DD`를 표시한다.
- FE-REQ-003-1: `종료` 단계 프로젝트는 프로그레스바 위 오버레이로 `종료 단계 · 종료일 YYYY.MM.DD`를 표시한다.
- FE-REQ-003-2: `검토/종료` 오버레이는 **가독성을 위해 크기를 크게** 표시하고, 배경은 **반투명(글래스) 스타일**로 처리한다. 오버레이가 커져도 상/하단 정보(예산/날짜 영역)와 겹치지 않아야 한다.
- FE-REQ-004: 일정 데이터가 로딩 중이거나 비어있는 경우 UI는 깨지지 않아야 하며, 플레이스홀더(`...` 또는 `-`)로 표시한다.

## 수용 기준(AC)
1. `/home`에서 프로젝트 리스트 카드의 프로그레스바가 4단계로 분리되어 표시된다.
2. 일정관리에서 설정한 설계/제작/설치 일정이 메인 리스트에 날짜로 표시된다.
3. 설치 종료일을 기준으로 AS 시작/종료(1년 후)가 계산되어 표시된다.
4. `검토` 단계 프로젝트는 오버레이로 검토 단계와 생성일자가 표시된다.
5. `종료` 단계 프로젝트는 오버레이로 종료 단계와 종료일자가 표시된다.
6. 오버레이가 커져도 카드 내 다른 정보(예산/날짜)와 겹치지 않는다.
7. `npm run verify:fast`가 통과한다.
