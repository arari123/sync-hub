# PRD: 이슈 관리 페이지 스크롤 위치 복원 (2026-02-15)

## 배경
이슈(안건) 목록에서 특정 이슈를 클릭해 상세로 이동한 뒤, 브라우저 뒤로가기로 목록 화면으로 돌아오면 스크롤 위치가 초기화된다. 목록을 탐색하던 사용자는 다시 스크롤을 내려 동일 지점을 찾아야 한다.

## 목표
1. 이슈 관리(안건 목록) 페이지에서 상세 화면으로 이동 후 뒤로가기로 복귀했을 때, 이전 상하 스크롤 위치가 유지되도록 한다.
2. 목록 데이터 로딩이 끝난 시점에 복원이 적용되어 "복원 실패"나 "위치 점프"가 최소화되도록 한다.

## 비목표
- 필터/검색/페이지네이션 상태를 URL로 영구 저장하는 개선은 이번 범위에서 제외한다.
- 다른 모든 리스트 페이지로의 일반화는 이번 범위에서 제외한다(필요 시 후속 작업으로 분리).

## 요구사항
### FE-REQ
- FE-REQ-001: 이슈 관리 페이지에서 다른 라우트로 이동하기 직전에 현재 `window.scrollY`를 저장해야 한다.
- FE-REQ-002: 상세에서 뒤로가기로 이슈 관리 페이지로 돌아올 때(History POP), 저장된 스크롤 위치로 자동 이동해야 한다.
- FE-REQ-003: 스크롤 복원은 목록 로딩이 끝난 이후(`isLoading === false`)에 수행되어야 한다.

## 수용 기준(AC)
1. `/project-management/projects/:projectId/agenda`에서 스크롤을 내린 뒤 임의 이슈 상세로 진입하고, 브라우저 뒤로가기로 돌아오면 이전 스크롤 위치가 유지된다.
2. `npm run verify:fast`가 통과한다.
