# PRD: 경영진 보고용 기획서(PPTX) 자동 생성 (2026-02-19)

## 배경
- 현재 구현된 Sync-Hub 웹페이지 전반의 기능을 경영진에게 한 번에 설명할 수 있는 시각화 자료가 필요하다.
- 이미 생성된 유형별 데모 프로젝트(설비/파츠/AS)를 기반으로 실제 화면을 포함한 신뢰 가능한 보고서가 필요하다.

## 목표
1. 현재 구현된 주요 웹페이지의 기능 설명을 빠짐없이 포함한다.
2. 유형별 프로젝트 시나리오(설비/파츠/AS)와 현재 데이터 상태를 함께 제시한다.
3. 실제 구현 화면 스크린샷을 포함한 경영진 보고용 PPTX를 산출한다.
4. 동일 산출물을 재생성할 수 있도록 스크립트 기반으로 구성한다.

## 요구사항
### DOC-REQ
- DOC-REQ-001: 보고서는 사용자가 전달한 기획의도/AS-IS vs TO-BE 내용을 반영해야 한다.
- DOC-REQ-002: 현재 구현된 웹페이지 기능 설명(라우트/기능 요약)을 포함해야 한다.
- DOC-REQ-003: 유형별 프로젝트 시나리오(설비/파츠/AS)와 대표 데이터 지표를 포함해야 한다.

### IMG-REQ
- IMG-REQ-001: 실제 실행 중인 화면을 캡처해 PPT에 삽입한다.
- IMG-REQ-002: 인증/홈/프로젝트/예산/안건/일정/데이터관리/데이터허브/사양(placeholder) 화면이 포함되어야 한다.

### AUTO-REQ
- AUTO-REQ-001: DB 컨텍스트 추출 스크립트를 제공한다.
- AUTO-REQ-002: 화면 캡처 스크립트를 제공한다.
- AUTO-REQ-003: PPTX 생성 스크립트를 제공한다.

## 수용 기준(AC)
1. `reports/executive/2026-02-19/SyncHub_경영진_기획보고서_2026-02-19.pptx` 파일이 생성된다.
2. 보고서에 사용자 제공 기획의도/AS-IS vs TO-BE가 포함된다.
3. 보고서에 웹페이지 기능 인벤토리(기능 설명)와 실제 캡처 이미지가 포함된다.
4. 보고서에 유형별 프로젝트 시나리오(설비/파츠/AS)가 포함된다.
5. Docker 환경에서 `verify:fast`가 통과한다.
