# PRD: 안건 관리 리스트 최초/최신 답변 2분할 카드 (2026-02-15)

## 1. 배경/문제
- 답변이 등록된 안건도 리스트에서 최초 등록 내용과 최신 답변 내용을 한 번에 확인하기 어렵다.
- “최초 안건 vs 최신 답변 안건”을 동시에 비교할 수 있도록 카드(패널) 자체가 좌/우로 명확히 분리되어야 한다.

## 2. 목표
- 이슈(안건) 관리 페이지의 리스트 카드에서, 답변이 존재하는 안건을 **정확히 50:50**으로 나누어
  - 왼쪽: 최초 등록 안건
  - 오른쪽: 해당 최신 답변 안건
  을 동시에 표시한다.

## 3. 범위
- 대상 페이지: 프로젝트 > 안건 관리(`/project-management/projects/:projectId/agenda`)
- 대상 데이터: `record_status=published` 이면서 `thread_kind=general` 이고 `reply_count > 0` 인 안건
- 비범위:
  - 안건 상세/작성/답변작성 페이지의 버튼/확인창(이번 작업에서 변경하지 않음)
  - 작업보고서(`thread_kind=work_report`) 카드 레이아웃 변경

## 4. 요구사항
- REQ-001: 답변이 존재하는 일반 안건(`reply_count > 0`)은 리스트 카드가 좌/우 2패널(50:50)로 표시되어야 한다.
- REQ-002: 왼쪽(최초 등록 안건) 패널에는 아래 정보가 표시되어야 한다.
  - 제목(최초 등록 제목)
  - 작성자
  - 요청자
  - 답변자(최초 등록 안건 기준)
- REQ-003: 오른쪽(최신 답변 안건) 패널에는 아래 정보가 표시되어야 한다.
  - 제목(최신 답변 제목)
  - 작성자(최신 답변 작성자)
  - 답변자(최신 답변 기준)
- REQ-004: 두 패널은 육안으로 확실히 구분되어야 한다.
  - 패널 라벨(예: “최초 등록 안건”, “최신 답변 안건”)
  - 패널 간 구분선
  - 서로 다른 배경 톤(또는 그라데이션) 등
- REQ-005: 좌/우 패널 모두 썸네일 이미지를 표시해야 한다.
  - 썸네일은 **제목 왼쪽**에 배치한다.
  - 크기는 “답변이 없는 안건 리스트”의 썸네일과 동일해야 한다.
- REQ-006: 안건 리스트 카드의 세로폭(높이)은 **답변 유무와 관계없이 동일**해야 한다.
  - 답변이 있는 카드가 추가 요소로 인해 더 높아지지 않도록 레이아웃/라인클램프를 조정한다.

## 5. API 변경
- `GET /agenda/projects/{project_id}/threads` 응답의 각 `item`에 아래 필드를 추가한다.
  - `root_summary_plain`: 최초 등록 본문 기반 요약(최대 600자)
  - `latest_summary_plain`: 최신 답변 본문 기반 요약(최대 600자)
  - `root_responder_name`, `root_responder_org`: 최초 등록 안건 기준 답변자 정보

## 6. 수용 기준(AC)
- 답변이 1개 이상인 일반 안건은 리스트에서 카드가 좌/우 2분할로 표시된다.
- 좌 패널에는 작성자/요청자/답변자가 모두 노출된다.
- 우 패널에는 작성자/답변자가 노출된다.
- 좌/우 패널이 배경/라벨/구분선으로 명확히 구분된다.
- 좌/우 패널 모두 썸네일이 제목 왼쪽에 표시된다.
- 답변 유무와 무관하게 리스트 카드의 높이가 동일하다.
