# PRD: 데이터 허브 AI 사용량 표시/무근거 응답 UX 개선 (2026-02-19)

## 배경
호스팅 환경에서 데이터 허브 AI 답변 생성 시, 근거 문서가 없는 경우 `usage={}` 응답으로 인해 UI에 `prompt - · output -`가 노출되어 기능 오동작처럼 보이는 문제가 발생했다.

## 문제 정의
- 무근거(no-context) 또는 안건 미일치(not-found) 응답에서 토큰 사용량이 없는데도, 프론트가 빈 객체를 사용량 데이터로 인식해 `-` 값을 표시한다.
- 사용자는 AI 호출 실패로 오해한다.

## 요구사항
1. 무근거/안건 미일치 응답에서 토큰 사용량은 `null`로 반환한다.
2. RAG 답변 성공 응답에는 모드(`rag_answer`)를 명시한다.
3. 프론트는 유효 토큰 수치가 있을 때만 `prompt/output` 사용량 문구를 노출한다.
4. 무근거 응답(`rag_no_context`)에서는 모델 호출 생략 안내 문구를 표시한다.

## 비범위
- Cloud Run에 Elasticsearch를 상시 구성하는 아키텍처 변경
- 무근거 상황에서 일반 생성형 답변으로 우회 호출하는 정책 변경

## 완료 기준
- 호스팅/로컬 모두에서 no-context 응답 시 `prompt - · output -` 문구가 노출되지 않는다.
- no-context 응답 시 안내 문구가 표시된다.
- RAG 성공 응답의 사용량 문구는 기존처럼 정상 표시된다.
