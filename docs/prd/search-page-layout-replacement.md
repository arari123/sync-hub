# PRD: 검색 페이지 레이아웃 실구현 교체

## 목적
`docs/code.html` 시안을 참고하여 실제 프론트엔드 검색 페이지(`/search`)를 대체한다. 디자인만 시안 형태로 교체하되 검색/프로젝트 데이터는 기존 백엔드 API와 연동 상태를 유지한다.

## 요구사항

### SPR-REQ-001: 검색 페이지 실구현 교체
- [AC-1] `docs/code.html`은 참고 자료로만 사용하고 실제 화면은 `frontend` 코드에 구현한다.
- [AC-2] 검색 페이지에서 기존 검색 동작(문서 검색 API 연동)이 유지되어야 한다.
- [AC-3] 프로젝트 목록은 실제 생성된 프로젝트 데이터가 표시되어야 한다.

### SPR-REQ-002: 레이아웃/카피 반영
- [AC-1] `프로젝트 요약 현황` 패널은 제거한다.
- [AC-2] `글로벌 검색 정책` 패널은 제거한다.
- [AC-3] `강력 필터링`, `(초기 접힘 상태)`, `공통 레이아웃`, `사이드바 (페이지별 변경)` 문구는 제거한다.
- [AC-4] 프로젝트 테이블의 `LATEST ISSUE` 문구를 `마지막 안건`으로 변경한다.
- [AC-5] `고객사/위치` 열 오른쪽에 `담당자` 열을 추가한다.
- [AC-6] 프로젝트 이름 왼쪽에 프로젝트 대표 사진(썸네일)을 표시한다.
- [AC-7] `새 프로젝트 생성` 버튼은 유지한다.

### SPR-REQ-003: 상단 구성
- [AC-1] 검색 페이지 상단은 로고, 검색 입력, 알림, 유저 아이콘 중심으로 구성한다.
- [AC-2] 페이지 내 `내 프로젝트/전체 프로젝트` 토글은 유지한다.

## 범위
- `frontend/src/pages/SearchResults.jsx` 개편
- `frontend/src/components/Layout.jsx` 검색 페이지 전용 레이아웃 조정
- PRD/실행계획 문서 추가

## 비범위
- 이슈 등록 기능 구현
- 프로젝트 안건 데이터 모델 신규 추가
- 백엔드 검색 스코어 알고리즘 변경
