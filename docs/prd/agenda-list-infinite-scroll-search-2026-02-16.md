# PRD: 안건 리스트 인피니티 스크롤/검색 단순화 개편 (2026-02-16)

## 목적
- 메인 `전체안건`과 프로젝트 `안건 관리` 리스트의 탐색 효율을 높이기 위해 페이지네이션을 제거하고 인피니티 스크롤로 전환한다.
- 리스트 상단 필터 버튼을 제거하고 검색창 중심 인터페이스로 단순화한다.
- 리스트 항목 정보를 핵심 메타데이터 중심(최초/답변, 제목, 작성자, 등록일)으로 정리해 가독성을 높인다.

## 적용 범위
- 프론트엔드 공용 컴포넌트: `AgendaSplitView`
- 메인 페이지 `전체안건` 탭(`SearchResults.jsx`)
- 프로젝트 `안건 관리` 페이지(`AgendaList.jsx`)
- 백엔드 엔트리 목록 API 검색 파라미터 확장

## 요구사항
- REQ-001: 안건 리스트는 페이지 번호 방식이 아니라 인피니티 스크롤 방식이어야 한다.
  - 최초 10개 로드
  - 리스트 하단 접근 시 10개씩 추가 로드
- REQ-002: 리스트 패널 높이는 패널 하단까지 안건 항목이 표시되도록 충분히 확보되어야 한다.
- REQ-003: 리스트 상단의 상태/유형/임시저장 필터 버튼은 제거한다.
  - 검색 입력창만 유지한다.
- REQ-004: 리스트 항목에서 본문 요약은 표시하지 않는다.
  - `최초/답변 구분`, `제목`, `작성자`, `등록일`만 노출한다.
- REQ-005: 작성자/등록일은 기존 대비 가시성이 높은 타이포 스타일을 사용한다.
- REQ-006: 정렬 우선순위는 최신 업데이트 안건(엔트리 `updated_at` 내림차순)을 유지한다.

## 수용 기준 (AC)
- AC-001: 메인/프로젝트 안건 리스트 모두에서 스크롤 하단 도달 시 10개씩 연속 로드된다.
- AC-002: 메인/프로젝트 안건 리스트 상단에 필터 버튼이 보이지 않고 검색창만 보인다.
- AC-003: 리스트 항목에 본문 텍스트가 노출되지 않으며, 요청한 4개 정보만 보인다.
- AC-004: 작성자/등록일 정보가 기존 대비 식별하기 쉬운 스타일로 표시된다.
- AC-005: 검색어 입력 시 메인/프로젝트 리스트 모두 동일하게 필터링된다.
