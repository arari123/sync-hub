# PRD: 글로벌 검색 + 프로젝트 모니터링 레이아웃

## 목적
향후 모든 Sync-Hub 페이지에 공통 적용할 상단 레이아웃(로고/글로벌 검색/알림/유저)을 확정하고, 프로젝트 모니터링 중심의 기본 화면(`docs/code.html`)을 정의한다.

## 요구사항

### GSPM-REQ-001: 공통 상단 레이아웃
- [AC-1] 상단바에는 `sync-hub` 로고, 글로벌 검색 입력, 알림 아이콘, 유저 아이콘만 배치한다.
- [AC-2] 기존 `내 프로젝트/전체 프로젝트` 전환 버튼은 상단바에서 제거한다.
- [AC-3] 향후 페이지에서도 글로벌 검색바 형태를 동일하게 유지한다.

### GSPM-REQ-002: 프로젝트 요약/목록 모니터링
- [AC-1] 검색창 아래에 프로젝트 요약 현황을 모니터링할 수 있는 요약 카드 영역을 제공한다.
- [AC-2] 프로젝트 리스트 영역에 `새 프로젝트 생성` 버튼을 포함한다.
- [AC-3] 기본 노출은 내 프로젝트 기준으로 하되, 내/전체 프로젝트 토글을 제공한다.
- [AC-4] 한 화면에서 10개 프로젝트를 리스트로 표시한다.

### GSPM-REQ-003: Latest Issue 열 상태
- [AC-1] `docs/code.html`의 `Latest Issue` 열은 전 항목 `Empty` 상태로 렌더링한다.
- [AC-2] 이슈 등록 기능 미구현임을 사용자에게 명확히 표시한다.

### GSPM-REQ-004: 글로벌 검색 정책 안내
- [AC-1] 글로벌 검색은 프로젝트 데이터 + 업로드 자료(PDF/EXCEL) 검색 가능 정책을 안내한다.
- [AC-2] 자연어 검색 지원을 명시한다.
- [AC-3] 결과 노출 우선순위를 `프로젝트 정보 > 안건 > 사양 > 자료 > UNIT > PART'S`로 명시한다.
- [AC-4] `프로젝트코드:삼성전자` 형태의 프로젝트 범위 검색 명령어 예시를 제공한다.

### GSPM-REQ-005: 필터링 UX
- [AC-1] 강력한 필터링 패널을 제공한다.
- [AC-2] 초기 진입 시 필터링 패널은 접힌 상태여야 하며 필요 시 펼쳐 사용할 수 있어야 한다.
- [AC-3] 프로젝트 업데이트(안건/예산/사양/기본정보) 항목은 강조 표시 및 바로가기 링크를 제공한다.

## 범위
- `docs/code.html` 레이아웃/카피 개편
- PRD/실행계획 문서 추가

## 비범위
- 실제 검색 인덱싱/랭킹 알고리즘 백엔드 구현
- 실제 이슈 등록/수정 기능 구현
- 필터링 서버 연동
