# PRD: 일정 작성 명칭 공백 입력 보존 수정 (2026-02-18)

## 배경
- 일정 작성 화면에서 명칭 입력 시 스페이스가 즉시 사라져 단어 사이 공백 입력이 어렵다.
- 입력 중 상태 정규화 과정에서 명칭 값이 매번 `trim()` 처리되어 발생한다.

## 목표
- 일정 작성에서 그룹/행 명칭 입력 시 사용자가 입력한 공백(특히 단어 사이 공백)이 입력 단계에서 유지되게 한다.

## 범위
- 프론트엔드
  - 일정 정규화 유틸의 명칭 필드 처리 방식 수정

## 비범위
- 일정 API의 저장 정책(서버측 strip/정규화 정책)
- 일정 관리 조회 화면의 필터/정렬 UX 개편

## 요구사항
- REQ-001: `frontend/src/lib/scheduleUtils.js`의 그룹/행 명칭 정규화는 실시간 `trim()`을 적용하지 않는다.
- REQ-002: 그룹 명칭이 공백만 입력된 경우에는 기존 기본값(`그룹`) 보정 동작을 유지한다.
- REQ-003: 기존 빠른 검증(`verify:fast`)이 Docker 환경에서 통과해야 한다.

## 완료 기준
- 일정 작성 화면에서 명칭 입력 중 스페이스가 즉시 제거되지 않는다.
- 로컬 Docker 환경에서 `verify:fast` 통과
