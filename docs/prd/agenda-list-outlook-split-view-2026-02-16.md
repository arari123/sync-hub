# PRD: 안건 리스트 아웃룩형(Split View) 개편 (메인/이슈관리 공통) (2026-02-16)

## 목적
- 안건을 “왼쪽 리스트 + 오른쪽 상세(본문)”의 아웃룩(Outlook) 유사 인터페이스로 개편해, 목록 탐색과 내용 확인을 빠르게 한다.
- 메인 페이지의 `전체안건`과 프로젝트 내부 `이슈 관리`(안건 관리) 페이지 모두 동일한 패턴으로 동작하도록 통일한다.
- 조회(읽음) 여부를 리스트에서 가시성 좋게 표현하고, 리스트 영역을 접고/펼칠 수 있게 한다.
- 스레드(최초등록) 단위가 아니라 “답변(엔트리)”도 별도의 리스트 항목으로 노출해 최신 응답을 바로 확인할 수 있게 한다.

## 적용 범위
- 메인 페이지(`/home`)의 `전체안건` 탭
- 프로젝트 페이지(`/project-management/projects/:projectId/agenda`)의 이슈 관리(안건) 목록
- 백엔드: 엔트리(최초등록/답변) 단위 목록 API 추가
- 조회 여부(읽음)는 1차로 브라우저 localStorage 기반으로 구현(서버 DB 스키마 변경 없음)

## 요구사항
- REQ-001: 화면 레이아웃은 Split View 형태여야 한다.
  - 왼쪽: 안건 리스트
  - 오른쪽: 선택된 리스트 항목의 본문/상세
- REQ-002: 왼쪽 리스트는 접기/펼치기 토글이 있어야 한다.
  - 접었을 때 오른쪽 상세 영역이 넓어져야 한다.
- REQ-003: 왼쪽 리스트 항목은 조회(읽음) 여부를 가시성 좋게 표현해야 한다.
  - 예: 미조회 뱃지/점, 볼드 타이포, 배경 강조 등
- REQ-004: 왼쪽 리스트는 “최초등록 안건(루트 엔트리)” 뿐 아니라 “답변 안건(리플라이 엔트리)”도 각각 별도의 리스트 항목으로 표시한다.
  - 정렬 기본값은 최신 업데이트(엔트리 updated_at) 내림차순
- REQ-005: 리스트에서 “답변 안건” 항목을 선택하면, 오른쪽 상세는 다음을 충족해야 한다.
  - 선택된 답변 엔트리의 본문이 상단에 표시된다.
  - 현재 안건 상세 페이지와 동일하게 하단에 “최초 등록 안건(루트 엔트리)”도 함께 표시된다.
- REQ-006: 메인 페이지 `전체안건`은 다음 범위를 포함한다.
  - 내가 담당자인 프로젝트(프로젝트 manager) 소속의 게시(published) 엔트리(루트/답변)
  - 내가 작성한 엔트리(루트/답변)는 프로젝트 담당 여부와 무관하게 포함
  - 내 draft(임시 저장) 루트 엔트리는 포함(다른 사용자의 draft는 제외)
- REQ-007: 프로젝트 `이슈 관리`는 해당 프로젝트에 소속된 엔트리(루트/답변)를 표시한다.
  - 옵션으로 “내 draft 포함” 토글을 제공할 수 있다.

## 조회(읽음) 정의
- 엔트리 단위로 관리한다.
  - 기준값: 엔트리의 `updated_at`
  - localStorage에 `{ entryId: last_seen_updated_at }` 형태로 저장한다.
- 사용자가 리스트 항목을 선택해 오른쪽 상세가 표시되면, 해당 엔트리를 “조회(읽음)” 처리한다.

## 수용 기준(AC)
- AC-001: 메인 `전체안건`과 프로젝트 `이슈 관리`에서 Split View 레이아웃이 적용된다.
- AC-002: 왼쪽 리스트 접기/펼치기가 동작한다.
- AC-003: 답변 엔트리가 별도의 리스트 항목으로 노출되고, 클릭 시 답변 본문 + 루트 본문이 함께 보인다.
- AC-004: 미조회 엔트리가 리스트에서 명확히 강조된다.
