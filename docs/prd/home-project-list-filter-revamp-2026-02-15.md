# PRD: 메인 프로젝트 리스트 필터 개편 (유형 → 단계)

- 작성일: 2026-02-15
- 적용 화면: 메인(`/home`) 프로젝트 리스트(검색어 `q` 없는 상태)

## 배경/문제
- 현재 메인 페이지에서 프로젝트 단계 필터가 항상 노출되어 있으며, 파츠/AS 프로젝트는 실제로 `검토-시작-종료`만 의미가 있는데도 설비 단계(설계/제작/설치/워런티)가 섞여 UX 혼란이 발생한다.

## 목표
1. 프로젝트 유형 필터를 우선 선택하게 하고, 선택된 유형에 맞는 단계 필터만 노출한다.
2. 파츠/AS는 `검토-시작-종료` 단계로만 필터링할 수 있어야 한다.

## 요구사항
### UI/배치
- 프로젝트 유형 버튼 그룹이 프로젝트 단계 버튼 그룹보다 왼쪽(선행)으로 배치된다.
- 프로젝트 단계 버튼은 초기에는 접힌(숨김) 상태다.
- 프로젝트 유형에서 `설비/파츠/AS` 중 하나를 선택하면 해당 유형에 맞는 단계 필터가 펼쳐진다.

### 유형별 단계 필터
- 설비(`equipment`): `검토-설계-제작-설치-워런티-종료`
- 파츠(`parts`): `검토-시작-종료`
- AS(`as`): `검토-시작-종료`

### 동작 규칙
- 유형 선택은 단계 필터 구성에 직접 영향을 주므로, 유형 필터는 단일 선택(재클릭 시 해제)으로 동작한다.
- 파츠/AS의 `시작`은 내부 stage 코드가 `review/closure`가 아닌 모든 상태를 포함하는 버킷으로 동작한다.

## 비목표
- 검색 결과 화면(`/home?q=...`)의 글로벌 검색 UX 개편은 이번 범위가 아니다.
