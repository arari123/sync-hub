# PRD: 상단바 사용자 메뉴(정보/로그아웃) (2026-02-15)

## 배경
현재 상단바의 사용자 아이콘(이니셜 배지)은 클릭해도 동작이 없다. 사용자는 Google 계정 메뉴처럼 클릭 시 **사용자 정보**를 확인하고 **로그아웃**할 수 있기를 원한다.

## 목표
1. 상단바 사용자 아이콘 클릭 시 드롭다운(팝오버) 메뉴를 표시한다.
2. 메뉴에 사용자 정보(이름/이메일)를 표시한다.
3. 메뉴에 로그아웃 액션을 제공한다.
4. 시각적 스타일은 Google 사용자 메뉴와 유사한 “카드형 팝오버” 느낌으로 구성한다.

## 비목표
- 사용자 프로필 편집/설정 페이지 신규 개발은 범위 밖이다.
- 멀티 계정 전환 기능은 범위 밖이다.

## 요구사항
### FE-REQ
- FE-REQ-001: 사용자 아이콘을 클릭하면 메뉴가 열리고, 외부 클릭 시 닫혀야 한다.
- FE-REQ-002: 메뉴 상단에 사용자 이름/이메일이 표시되어야 한다.
- FE-REQ-003: 메뉴에 `로그아웃` 버튼이 있어야 하며, 클릭 시 세션이 종료되고 `/login`으로 이동해야 한다.
- FE-REQ-004: 로그아웃 요청이 실패하더라도(네트워크 오류 등) 로컬 세션은 정리되고 로그인 화면으로 이동해야 한다.

### BE-REQ
- BE-REQ-001: 기존 `POST /auth/logout`을 사용한다(추가 API 불필요).

## 수용 기준(AC)
1. `/home` 및 GlobalTopBar가 노출되는 라우트에서 사용자 아이콘 클릭 시 사용자 메뉴가 표시된다.
2. 메뉴에 이름/이메일이 표시되고, `로그아웃`이 동작한다.
3. `npm run verify:fast`가 통과한다.
