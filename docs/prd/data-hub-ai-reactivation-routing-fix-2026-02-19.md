# PRD: 데이터 허브 AI 재활성화 - 호스팅 API 경로 충돌 수정 (2026-02-19)

## 배경
- 데이터 허브 페이지에서 AI 답변 생성 버튼이 비활성화되어 있다.
- 원인 분석 결과, 호스팅에서 `/data-hub`는 프론트 라우트인데 API도 `/data-hub/*`를 사용해 권한 조회가 실패한다.

## 목표
1. 호스팅에서 데이터 허브 API 호출 경로를 프론트 라우트와 충돌하지 않도록 수정한다.
2. 데이터 허브 페이지의 권한 조회/AI 질의/업로드가 정상 라우팅되도록 복구한다.
3. Gemini 모델 기본값은 `gemini-2.5-flash-lite`를 유지한다.

## 요구사항
- REQ-001: 백엔드는 `/api/data-hub/*` 경로로도 동일 API를 제공해야 한다.
- REQ-002: 프론트 데이터 허브는 `/api/data-hub/*` 경로로 API를 호출해야 한다.
- REQ-003: Docker 환경에서 `verify_fast`, 프론트 `build`가 통과해야 한다.

## 수용 기준
1. 호스팅 환경에서 데이터 허브 권한 API가 프론트 HTML이 아닌 백엔드 JSON을 반환한다.
2. 데이터 허브 페이지에서 AI 관련 API 호출 경로가 정상 동작한다.
3. `verify_fast` 및 `npm run build`가 통과한다.
